4. ФУНКЦИОНАЛЬНЫЕ МОДУЛИ

4.1. Модуль каталога товаров:

4.1.1. Описание:
Каталог товаров обеспечивает навигацию по товарам, фильтрацию, сортировку и поиск. Поддерживает иерархическую структуру категорий.

4.1.2. Функциональность:
— Отображение списка товаров с пагинацией;
— Фильтрация по параметрам: категория, бренд, цена, наличие, характеристики;
— Сортировка: по цене, по популярности, по рейтингу, по новизне;
— Поиск товаров (интеграция с Meilisearch/Typesense — опционально);
— Карточка товара: изображения, название, артикул, цена, старая цена, скидка, наличие, характеристики, отзывы;
— Хлебные крошки (breadcrumbs) для навигации;
— SSR-рендеринг для SEO-оптимизации.

4.1.3. Модели данных:
— Product: id, name, slug, description, price, oldPrice, discount, stock, rating, categoryId, brandId, images[], characteristics[], parserPrice, parserSources, useParserPrice и др.;
— Category: id, name, slug, parentId, order, icon;
— Brand: id, name, slug, logo, description.

---

4.2. Модуль парсинга цен:

4.2.1. Описание:
Автоматизированная система сбора и обновления цен с маркетплейса Wildberries. Состоит из сервера парсинга (parser-server), Chrome-расширения и интеграции с основным приложением.

4.2.2. Функциональность:
— Chrome-расширение: пакетный парсинг товаров по URL (извлечение nmId, получение данных через WB API);
— Отправка результатов парсинга webhook на сервер (http://<server>/api/admin/parser/webhook);
— Сервер парсинга (parser-server): Node.js + Express сервер (порт 3005), API endpoints;
— Приоритетное определение цены: несколько источников с приоритетами (0 = главный), перебор по порядку, проверка наличия;
— Автоматическое обновление цены товара: при useParserPrice = true цена обновляется автоматически;
— История цен (PriceHistory): все изменения цен с указанием причины;
— Логи парсинга: хранение всех задач парсинга со статусами, фильтрация и пагинация;
— Админ-интерфейс: вкладка «Парсинг» в редактировании товара (добавление источников, приоритеты, запуск парсинга);
— Страница логов парсинга в админке (/admin/parser/logs).

4.2.3. Модели данных:
— Product.parserSources: JSON массив [{url, priority, isActive}];
— Product.parserPrice, parserOldPrice, parserUpdatedAt, parserInStock, parserDelivery, parserName;
— Product.useParserPrice: Boolean — флаг использования цен с парсера;
— PriceHistory: id, productId, oldPrice, newPrice, reason («parser_auto»), createdAt;
— ParserLog: id, taskId, productId, status, url, result, error, createdAt.

4.2.4. Chrome-расширение (extension/):
— manifest.json (Manifest V3);
— popup.html / popup.js — интерфейс пакетного парсинга;
— background.js — service worker;
— tracker.html / tracker.js — трекер процесса парсинга;
— permissions: storage, tabs, notifications, alarms, scripting;
— host_permissions: https://*.wildberries.ru/*, http://localhost:3002/*.

4.3. Модуль конфигуратора ПК:

4.3.1. Описание:
Интерактивный конфигуратор позволяет пользователю собрать собственный ПК из совместимых комплектующих по категориям.

4.3.2. Функциональность:
— Выбор комплектующих по категориям (CPU, GPU, Motherboard, RAM, Storage, PSU, Case, Cooling и др.);
— Проверка совместимости компонентов (сокет CPU и motherboard, форм-фактор, TDP, объём RAM и др.);
— Расчёт требуемой мощности блока питания (TDP);
— Отображение итоговой стоимости и характеристик сборки;
— Сохранение сборки в профиль пользователя;
— Загрузка сохранённой сборки;
— Публикация сборки (shareCode) — опционально;
— Пресеты сборок (готовые конфигурации).

4.3.3. Модели данных:
— Configuration: id, userId, name, isPreset, total, power, isPublic, shareCode, createdAt;
— ConfigItem: id, configId, categoryId, productId, quantity, price.

4.4. Модуль корзины и оформления заказа:

4.4.1. Описание:
Корзина позволяет добавлять товары, изменять количество, переходить к оформлению заказа.

4.4.2. Функциональность:
— Добавление/удаление товаров из корзины;
— Изменение количества товаров;
— Расчёт итоговой стоимости;
— Корзина сохраняется в базе данных (привязка к пользователю);
— Оформление заказа: выбор адреса доставки, комментарий;
— Создание заказа в базе данных;
— Генерация номера заказа (orderNumber).

4.4.3. Модели данных:
— Cart: id, userId;
— CartItem: id, cartId, productId, quantity;
— Order: id, userId, orderNumber, status, paymentStatus, total, deliveryAddress, comment, createdAt;
— OrderItem: id, orderId, productId, quantity, price.

Список желаний — позволяет пользователю сохранять понравившиеся товары.

4.5.3. Модели данных:

4.7. Модуль аутентификации и профилей пользователей:

4.7.4. История заказов:

— Фильтрация по статусу;
— Детали заказа: состав, статус, сумма, дата;
— Пагинация списка заказов.

4.7.5. Модели данных:
— User: id, email, password, name, phone, avatar, role, bonusPoints, level, emailVerified, createdAt;
— Address: id, userId, name, city, street, building, apartment, postalCode, phone, isDefault.

10. ТРЕБОВАНИЯ К БЕЗОПАСНОСТИ

10.1. Аутентификация:
— JWT-токены в httpOnly cookies (недоступны из JavaScript);
— Время жизни токена: 24 часа;
— Автоматическое продление сессии при истечении < 2 часов;
— Хеширование паролей: bcryptjs (10 salt rounds).

10.2. Заголовки безопасности (Next.js headers):
— Strict-Transport-Security (HSTS);
— X-Frame-Options (DENY);
— X-Content-Type-Options (nosniff);
— X-XSS-Protection;
— Referrer-Policy (strict-origin-when-cross-origin).

10.3. Роуты и авторизация:
— Middleware (middleware.ts) проверяет JWT-токен;
— /profile/* — только авторизованные;
— /admin/* — только ADMIN/MANAGER;
— Валидация всех входящих данных через Zod.

10.4. Инфраструктура:
— Non-root пользователь в Docker-контейнерах;
— SSL/TLS шифрование (Let's Encrypt);
— Rate limiting через Nginx;
— .dockerignore для исключения чувствительных файлов.


11. ТРЕБОВАНИЯ К РАЗВЁРТЫВАНИЮ

11.1. Локальная разработка:
— Docker Compose (app + postgres + parser + parser-ui);
— Команда: docker compose up -d;
— Порты: Next.js (3000), Parser (3005), Parser UI (3006), PostgreSQL (5432).

11.2. Продакшен:
— Docker Compose с multi-stage build;
— Nginx reverse proxy + SSL;
— Команда: docker compose -f docker-compose.prod.yml up -d;
— Применение миграций: npx prisma migrate deploy;
— Автозапуск через systemd.

11.3. CI/CD:
— GitHub Actions: CI pipeline (lint, tsc, test, build) при каждом push/PR;
— GitHub Actions: CD pipeline (деплой на Hetzner VPS) при merge в main;
— Требуемые секреты: VPS_HOST, VPS_USER, VPS_SSH_KEY, JWT_SECRET, COOKIE_SECRET, POSTGRES_PASSWORD.

11.4. Альтернативные платформы:
— Railway: автоматический деплой при push (railway.toml);
— Netlify: деплой фронтенда (netlify.toml);
