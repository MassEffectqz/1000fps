# 🔍 Диагностика проблем

## Быстрая проверка

### 1. Товар не добавляется

**Ошибка: "Нет открытых вкладок wildberries.ru"**

✅ **Решение:**

1. Откройте `wildberries.ru` в любой вкладке
2. Дождитесь загрузки страницы
3. Попробуйте добавить товар снова

---

**Ошибка: "Товар не найден"**

✅ **Решение:**

1. Проверьте правильность артикула
2. Убедитесь, что товар существует на WB
3. Нажмите ℹ️ в popup для проверки статуса

---

**Ошибка: "Цена не найдена"**

✅ **Решение:**

1. Откройте консоль Service Worker (см. ниже)
2. Посмотрите, какой ответ приходит от API
3. Возможно, товар недоступен в вашем регионе

---

### 2. Консоль Service Worker

**Как открыть:**

```
1. chrome://extensions/
2. Найдите "1000fps WB Парсер"
3. Нажмите "Service Worker" на карточке
4. Откроется консоль разработчика
```

**Что искать:**

```
[1000fps] Начинаем проверку 5 товаров...
[1000fps] Endpoint 0 error: HTTP 404
[1000fps] Переключение на endpoint #1
[1000fps] Успех через endpoint #1
```

---

### 3. Кнопка ℹ️ в popup

Нажмите **ℹ️** для просмотра статуса:

```
🔧 1000fps WB Парсер

📡 Endpoint'ов: 4
🚀 Рабочий API: #1
🌍 Dest: 123585942
⏱️ Кэш dest: 14:30:00
```

**Расшифровка:**

- `Endpoint'ов` — сколько запасных вариантов API
- `Рабочий API` — какой endpoint используется сейчас
- `Dest` — регион доставки (из кэша WB)
- `Кэш dest` — когда обновлялся кэш

### 4. Частые ошибки

#### "HTTP 403" или "HTTP 404"

**Причина:** WB заблокировал запрос

✅ **Решение:**

- Парсер автоматически переключится на другой endpoint
- Если не помогло — перезагрузите вкладку WB
- Проверьте, что вы не используете VPN/прокси

---

#### "Нет ответа от страницы WB"

**Причина:** Скрипт не выполнился в странице

✅ **Решение:**

1. Обновите вкладку WB
2. Убедитесь, что страница полностью загрузилась
3. Перезагрузите расширение

---

#### "Server sync error"

**Причина:** Сервер недоступен

✅ **Решение:**

1. Проверьте, что сервер запущен на `http://localhost:3001`
2. Проверьте логи сервера
3. Убедитесь, что брандмауэр не блокирует соединение

---

### 5. Сброс кэша

Если парсер работает некорректно:

```javascript
// В консоли Service Worker выполните:
chrome.storage.local.clear(() => location.reload());
```

Или просто перезагрузите расширение в `chrome://extensions/`.
