# 🔧 Адаптивность парсера к изменениям WB

## Что реализовано

### 1. Множественные API Endpoint'ы

Парсер использует **4 запасных варианта** для получения данных:

```javascript
1. wildberries.ru/__internal/card/cards/v4/detail  (основной)
2. wildberries.ru/__internal/card/cards/v3/detail  (запасной v3)
3. card.wb.ru/cards/detail                         (альтернативный домен)
4. catalog.wb.ru/cards/detail                      (ещё один вариант)
```

**Как работает:**

- При ошибке основного API → автоматически переключается на следующий
- Запоминает рабочий endpoint в кэше
- При следующем запуске начинает с рабочего endpoint

---

### 2. Обработка разных структур ответа

WB может возвращать данные в разных форматах. Парсер понимает:

```javascript
// Формат 1: data.data.products[0]
{ data: { products: [{ id, name, price... }] } }

// Формат 2: data.products[0]
{ products: [{ id, name, price... }] }

// Формат 3: data.card
{ card: { id, name, price... } }

// Формат 4: data (сам товар)
{ id, name, price... }
```

---

### 3. Адаптивное извлечение цены

Цена может быть в разных полях:

```javascript
// Формат 1: sizes[0].price.product/basic
{ sizes: [{ price: { product: 1927600, basic: 8116300 } }] }

// Формат 2: salePriceU / priceU
{ salePriceU: 1927600, priceU: 8116300 }

// Формат 3: price / oldPrice
{ price: 1927600, oldPrice: 8116300 }

// Формат 4: строки
{ salePrice: "19276.00", price: "81163.00" }
```

Парсер проверяет все варианты по очереди.

---

### 4. Кэширование параметров

**Кэшируется:**

- `dest` — регион доставки (из localStorage WB)
- `workingEndpoint` — индекс рабочего API
- `lastError` — последняя ошибка для отладки

**TTL кэша:** 5 минут

**Зачем:**

- Не запрашивать dest каждый раз (экономия времени)
- Быстрее переключаться при ошибках API
- Иметь историю для отладки

---

### 5. Поддержка разных CDN изображений

**Новый CDN (приоритетный):**

```
https://rst-basket-cdn-01bl.geobasket.ru/vol123/part123456/123456789/images/big/1.webp
```

**Старый CDN (запасной):**

```
https://basket-01.wbbasket.ru/vol123/part123456/123456789/images/big/1.jpg
```

Автоматически определяется по номеру товара.

## Что будет при изменениях WB

### Сценарий 1: Изменился API endpoint

✅ **Автоматически** переключится на запасной вариант  
✅ Продолжит работать через другой домен  
✅ Запомнит рабочий endpoint

---

### Сценарий 2: Изменилась структура ответа

✅ **Адаптивно** извлечёт данные из новой структуры  
✅ Проверит все известные форматы  
✅ Если не сработает → попробует следующий endpoint

---

### Сценарий 3: Изменился CDN картинок

✅ **Автоматически** переключится на старый формат  
✅ Картинки продолжат отображаться  
✅ Добавится новый формат в код

---

### Сценарий 4: Заблокировали запросы без авторизации

✅ Расширение работает **из контекста страницы WB**  
✅ Использует куки страницы автоматически  
✅ Не требует явных токенов

---

## Как добавить новый endpoint

Если WB изменит API, добавьте новый endpoint в `src/background.js`:

```javascript
const API_ENDPOINTS = [
  // ...существующие...

  // Новый endpoint:
  (article, dest) =>
    `https://new-api.wildberries.ru/cards?nm=${article}&dest=${dest}`,
];
```

## Диагностика проблем

### 1. Откройте консоль Service Worker

```
chrome://extensions/ → Service Worker (на карточке расширения)
```

### 2. Посмотрите логи

```
[1000fps] Endpoint 0 error: HTTP 404
[1000fps] Переключение на endpoint #1
[1000fps] Успех через endpoint #1
```

### 3. Нажмите ℹ️ в popup

Покажет текущий кэш и рабочий endpoint.

---

## Рекомендации

### При проблемах:

1. **Перезагрузите расширение** в `chrome://extensions/`
2. **Откройте вкладку WB** (требуется для работы)
3. **Проверьте консоль** Service Worker на ошибки
4. **Нажмите ℹ️** для просмотра статуса

### Для стабильности:

1. **Не закрывайте вкладку WB** (можно свернуть)
2. **Периодически обновляйте** расширение при изменениях WB
3. **Следите за логами** при массовом изменении цен
