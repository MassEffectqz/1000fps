# Система отзывов 1000FPS

## ✅ Реализовано

### 1. База данных
- ✅ Модель `Review` в Prisma схеме
- ✅ Уникальный индекс `[productId, userId]` - один отзыв на товар от пользователя
- ✅ Поля: rating, title, text, pros, cons, isVerified, isApproved, helpful, images
- ✅ Связи с Product и User

### 2. API Endpoints

#### Публичные:
- `GET /api/reviews` - получение отзывов с пагинацией и фильтрами
- `POST /api/reviews` - создание отзыва (требуется авторизация)
- `GET /api/reviews/[id]` - получение отзыва по ID
- `PUT /api/reviews/[id]` - обновление отзыва (только автор)
- `DELETE /api/reviews/[id]` - удаление отзыва (автор или админ)
- `POST /api/reviews/[id]/vote` - голосование за полезность

#### Админские:
- `GET /api/admin/reviews` - список отзывов для модерации
- `PUT /api/admin/reviews/[id]` - одобрить/отклонить отзыв
- `DELETE /api/admin/reviews/[id]` - удалить отзыв

### 3. Валидация (Zod)
- `createReviewSchema` - создание отзыва
- `updateReviewSchema` - обновление отзыва
- `voteReviewSchema` - голосование
- `moderateReviewSchema` - модерация
- `getReviewsSchema` - получение с фильтрами

### 4. Компоненты

#### Публичные:
- `ReviewsList` - список отзывов с:
  - Сводкой рейтинга (средний балл, количество)
  - Распределением по звездам (5-4-3-2-1)
  - Сортировкой (новые, положительные, отрицательные, полезные)
  - Карточками отзывов с:
    - Аватаром пользователя
    - Бейджем "Покупка подтверждена"
    - Рейтингом звездами
    - Плюсами и минусами
    - Голосованием за полезность

- `ReviewForm` - форма отзыва с:
  - Выбором рейтинга (1-5 звезд)
  - Заголовком (до 100 символов)
  - Текстом отзыва (10-2000 символов)
  - Преимуществами (до 500 символов)
  - Недостатками (до 500 символов)
  - Валидацией на клиенте
  - Индикатором отправки

#### Админские:
- `AdminReviewsPage` - страница модерации с:
  - Фильтрами (на модерации, одобренные, все)
  - Пагинацией
  - Кнопками одобрения/отклонения
  - Удалением отзывов
  - Отображением информации о товаре

### 6. Бизнес-логика

#### Создание отзыва:
1. Проверка авторизации
2. Валидация данных
3. Проверка существования товара
4. Проверка что пользователь не оставлял отзыв ранее
5. Проверка покупки товара (для verified статуса)
6. Создание отзыва со статусом `isApproved: false`
7. Уведомление о необходимости модерации

#### Модерация:
1. Только ADMIN и MANAGER могут модерировать
2. Одобрение/отклонение с причиной
3. Автоматическая публикация после одобрения

#### Голосование:
1. Только авторизованные пользователи
2. Нельзя голосовать за свой отзыв
3. Increment/decrement helpful count

## 🚀 Использование

### Добавление отзыва:
```typescript
POST /api/reviews
{
  "productId": "clx...",
  "rating": 5,
  "title": "Отличный товар!",
  "text": "Очень доволен покупкой...",
  "pros": "Быстрый, надежный",
  "cons": "Цена высоковата"
}
```

### Получение отзывов:
```typescript
GET /api/reviews?productId=clx...&page=1&limit=10&sortBy=newest
```

### Модерация:
```typescript
PUT /api/admin/reviews/clx...
{
  "isApproved": true
}
```
