# Страница добавления/редактирования товара

## Обзор

Создана полноценная система управления товарами в админ-панели Next.js с поддержкой всех необходимых функций для интернет-магазина электроники.

## Структура файлов

### Страницы
- `/src/app/admin/products/new/page.tsx` - страница добавления нового товара
- `/src/app/admin/products/[id]/page.tsx` - страница редактирования товара

### API Routes
- `/src/app/api/admin/products/route.ts` - GET (список), POST (создание)
- `/src/app/api/admin/products/[id]/route.ts` - GET (товар), PUT (обновление), DELETE (удаление)
- `/src/app/api/admin/categories/route.ts` - GET (список категорий), POST (создание)
- `/src/app/api/admin/brands/route.ts` - GET (список брендов), POST (создание)
- `/src/app/api/admin/warehouses/route.ts` - GET (список складов), POST (создание)
- `/src/app/api/admin/tags/route.ts` - GET (список тегов), POST (создание)
- `/src/app/api/admin/categories/[id]/specifications/route.ts` - GET/POST спецификации категории

### Компоненты
- `/src/components/ui/form.tsx` - базовые компоненты формы (Input, Textarea, Select, Checkbox, Card, Button, Label)
- `/src/components/ui/product-form.tsx` - специализированные компоненты:
  - `ProductImageUploader` - загрузка изображений с drag & drop
  - `TagInput` - ввод тегов с автодополнением
  - `ProductVariantEditor` - редактор вариантов товара
  - `SpecificationEditor` - редактор характеристик
- `/src/components/ui/product-form-container.tsx` - основной контейнер формы товара

### Валидация
- `/src/lib/validations/product.ts` - Zod схемы для валидации товара

### Server Actions
- `/src/lib/actions/product.ts` - серверные действия:
  - `createProduct` - создание товара
  - `updateProduct` - обновление товара
  - `deleteProduct` - удаление товара
  - `getProductById` - получение товара по ID

### Схема БД
- `/prisma/schema.prisma` - обновленная схема Prisma

## Обновления схемы Prisma

### Новые модели
- `ProductVariant` - варианты товара
- `Tag` - теги
- `ProductTag` - связь товаров и тегов
- `CategorySpecification` - спецификации для категорий

### Обновленные модели
- `Product` - добавлены поля:
  - `discountType`, `discountValue` - скидки
  - `weight`, `length`, `width`, `height` - габариты
  - `warrantyPeriod`, `warrantyType` - гарантия
  - `metaTitle`, `metaDescription`, `metaKeywords` - SEO
  - `isDraft` - черновик
- `ProductSpecification` - добавлено `isVariant`
- `Category` - добавлена связь `specifications`

### Новые enum
- `DiscountType` - PERCENT, FIXED
- `WarrantyType` - MANUFACTURER, SELLER
- `SpecType` - TEXT, NUMBER, SELECT, BOOLEAN

## Использование

### Создание товара
1. Перейти на `/admin/products/new`
2. Заполнить обязательные поля (название, категория, артикул, цена)
3. При необходимости заполнить дополнительные поля
4. Нажать "Создать товар"

### Редактирование товара
1. Перейти на `/admin/products/[id]`
2. Внести изменения
3. Нажать "Сохранить"
4. Или "Удалить товар" для удаления

### Работа с вариантами
1. Перейти на вкладку "Варианты"
2. Добавить варианты (цвет, размер и т.д.)
3. Указать цену и остаток для каждого варианта

### Работа со складами
1. Перейти на вкладку "Склады"
2. Указать количество для каждого склада
3. При необходимости указать зарезервированное количество

## Технические детали

### Стек
- Next.js 15 App Router
- TypeScript strict
- Prisma ORM
- PostgreSQL
- Zod валидация
- Tailwind CSS

### Особенности реализации
- Все API routes используют try/catch для обработки ошибок
- Параметризованные запросы Prisma (никакого сырого SQL)
- revalidatePath после мутаций для обновления кэша
- Транзакции для атомарных операций
- Singleton Prisma клиента в lib/prisma.ts

### Типы данных
Ценовые поля хранятся в копейках (Decimal), конвертируются на лету при отображении.
